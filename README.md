# 등기부등본 쉬운 해석기

방문자가 자기 **등기사항전부증명서(등기부등본) PDF**를 올리면, **브라우저 안에서만** 갑구·을구를 읽어
"언제 돈을 빌렸고, 언제 가압류·경매가 들어왔는지"를 **시간순 타임라인**으로 정리해 보여주는 무료 웹 도구입니다.
결과는 두 단계 — 먼저 **기본 해석**, 버튼 한 번으로 **중학생 해석**(집요정 캐릭터).

## 핵심 원칙
- **권리분석·법적판단 안 함.** 등기부에 적힌 사실(날짜·금액·이름·권리종류)만 정리하고, 용어는 사전적 설명만 덧붙입니다.
- **방문자 PDF·파싱결과는 어떤 경우에도 외부로 전송하지 않음.** 분석은 방문자 브라우저 안에서만 이뤄집니다(순수 클라이언트 사이드). 이 약속은 불변입니다.
- **외부 API·AI 호출·CDN 없음.** 단, 운영자가 방문 집계(GoatCounter)를 켜면 **익명 페이지뷰만** 전송됩니다(아래 "방문 집계" 참고). 끄면 외부통신 0. PDF·결과 미전송은 켬/끔과 무관하게 항상 참입니다.
- **용어 뜻풀이는 「법령용어한영사전(제2판), 법제처 2009」** 출처만 사용하며, 사전에 없는 용어는 "자료불충분"으로 표기합니다(임의 해석 금지).

## 방문자 사용법 (설치 불필요)
웹브라우저 하나면 됩니다. node·python 등 어떤 설치도 필요 없습니다.
1. **티스토리 해석기 글**로 접속(도구가 글 안에 들어 있습니다. github.io 주소로 직접 접속하면 안내 페이지가 뜹니다 — 아래 "도메인 잠금" 참고)
2. 등기부 PDF를 끌어다 놓거나 [파일 선택]
3. 진행바가 차오른 뒤(약 2초) 기본 해석 표시 → [쉽게 보기]로 중학생 해석

> 진행바의 약 2초는 파일을 읽는 동안 진행 상황을 보여주기 위한 표시입니다(분석은 방문자 브라우저 안에서만 이뤄지며 외부 전송은 없습니다). 티스토리 글에 iframe으로 실으면 내용 높이에 맞춰 iframe이 자동으로 늘어납니다(아래 배포 ② 참고).

> 인터넷등기소에서 받은 **'열람용'(텍스트가 들어있는) PDF**를 사용하세요. 이미지로 스캔된 PDF는 글자를 읽지 못합니다.

## 기술 구조
- HTML + CSS + Vanilla JS **단일 정적 페이지(무빌드)**.
- **PDF.js self-host**: `vendor/pdfjs/`에 동봉(외부 CDN 미사용) → 외부통신 0, CDN 장애에도 안 죽음.
- 모바일 우선 반응형(📱375 / 📲768 / 💻1280), OS 다크모드 대응.

### 폴더 구조
```
index.html              화면 골격 + 업로드 + 면책 + PDF.js 로컬 로드

js/parser.js            갑구·을구 파싱 → 사실 JSON (말소판정·표제부 포함)
js/glossary.js          용어집(층1 사건풀이 템플릿 / 층2 사전 뜻풀이)
js/render.js            기본/중학생 두 모드 렌더
js/mascot.js            집요정 SVG 캐릭터
js/app.js               흐름 제어(1회 파싱 → 두 모드 토글)
data/legal-terms.js     법령용어한영사전 발췌(빌드타임 생성, 출처표기)
vendor/pdfjs/           PDF.js self-host (.mjs 2개)
```

### 방문 집계 (운영자 전용, 선택)
방문 수는 **운영자만** GoatCounter 대시보드에서 확인합니다. **화면에는 카운터가 표시되지 않습니다**(방문자에겐 보이지 않음). **기본은 꺼져 있습니다.**

설정 방법:
1. [goatcounter.com](https://www.goatcounter.com)에서 무료 가입 → 사이트코드(예: `myblog`) 발급.
2. `index.html`의 `var GOATCOUNTER_CODE = ""` 에 본인 코드를 입력(예: `var GOATCOUNTER_CODE = "myblog";`).
3. 배포하면 그때부터 익명 방문이 집계됩니다. 대시보드는 `https://<코드>.goatcounter.com`.

프라이버시:
- 코드를 비워두면 집계하지 않고 **외부통신 0**을 유지합니다.
- 코드를 넣으면 **익명 페이지뷰만** GoatCounter로 전송됩니다(쿠키 없음, 개인정보·PDF·파싱결과 전송 없음).
- **방문자 PDF·파싱결과는 켬/끔과 무관하게 어떤 경우에도 전송되지 않습니다.**

유입 출처(네이버 vs 티스토리) 확인:
- 도구는 티스토리 글에 iframe으로 실리므로 GoatCounter가 보는 referrer는 대개 티스토리입니다. **네이버 vs 티스토리 유입 비교는 티스토리 자체 통계**(글의 유입경로)에서 확인하세요. GoatCounter는 **도구 총 방문 수** 확인용입니다.

## 배포 (GitHub Pages)
1. 이 폴더를 GitHub 저장소에 푸시(등기부 PDF는 `.gitignore`로 커밋 금지).
2. Settings → Pages → Branch `main` / 루트 `/` 선택 → 저장.
3. 발급된 `https://<user>.github.io/<repo>/` 주소로 접속해 동작 확인.
   - ⚠️ **도메인 잠금**이 켜져 있으면 직접 접속 시 안내 페이지가 뜹니다. 배포 후 동작 확인은 `?dev=키`를 붙여 접속하세요(아래 "도메인 잠금" 참고).
   - ⚠️ ES모듈·워커 특성상 **반드시 http(s)로 접속**해야 동작합니다. 로컬에서 파일을 더블클릭(`file://`)하면 안 됩니다. (로컬 테스트는 `python -m http.server` 등으로.)
   - ⚠️ **업데이트 배포 시 캐시 주의**: JS 모듈이 고정 파일명이라, 도구를 수정해 다시 올리면 재방문자 브라우저가 구버전 모듈을 캐시해 신/구가 섞일 수 있습니다(첫 방문은 무관). 업데이트할 때는 `index.html`의 `import "./js/app.js"` 등 모듈 경로 끝에 공통 버전 쿼리(예: `?v=2`)를 함께 올리거나, 안내에 "새로고침(Ctrl+F5)"을 덧붙이세요.

## 운영 구조 (유입·체류 퍼널)
도구는 **한 곳(GitHub Pages)에 원본 호스팅**하고, 블로그 2곳은 역할을 나눠 씁니다.

```
[네이버 블로그]  소개글 + "해석기 사용하기" 버튼/링크
      │  (클릭 → 티스토리 글로 이동. 네이버 본문엔 도구를 넣지 않음)
      ▼
[티스토리 글]   GitHub Pages 도구를 iframe으로 삽입 → 방문자가 글 안에서 PDF 해석
      │           (체류시간이 티스토리 글에 남음)   ← iframe src
      ▼
[GitHub Pages]  도구 정적 호스팅. PDF는 방문자 브라우저 안에서만 분석(외부전송 0)
```
- 네이버 블로그 본문은 스마트에디터가 iframe/embed 태그를 삭제하므로 **도구 직접삽입이 불가**합니다. 그래서 네이버는 **유입용(소개글+버튼)**만 담당하고, 클릭 시 **티스토리 해석기 글**로 보내 체류를 티스토리에 남깁니다.
- 이 퍼널은 **도메인 잠금**으로 강제됩니다: github.io 직접 접속은 안내 페이지, 실제 이용은 티스토리 글 안에서(아래 "도메인 잠금" 섹션).
- 아래 세 배포 경로는 **역할이 다르므로 혼동하지 마세요.**

## 배포 ② 티스토리 iframe 삽입 코드 (체류형, 최우선)
티스토리 글쓰기에서 **HTML 모드**로 전환한 뒤 아래를 그대로 붙여넣습니다.
```html
<iframe id="deungkiFrame" src="https://a01093126555-alt.github.io/platinum/?embed=1&from=tistory"
        style="width:100%;border:0;min-height:600px" loading="lazy"
        title="등기부등본 쉬운 해석기"></iframe>
<script>
addEventListener('message',function(e){
  if(e.data&&e.data.type==='deungki:height'){
    var f=document.getElementById('deungkiFrame');
    if(f&&e.data.height) f.style.height=(e.data.height+20)+'px';
  }
});
</script>
```
- `?embed=1` 이면 도구가 **임베드 모드**(큰 헤더를 숨겨 세로공간 절약)로 뜹니다. 면책·업로드 안내·결과는 그대로 노출됩니다.
- 붙여넣은 `<script>`는 도구가 보내는 **높이 값을 받아 iframe 높이를 자동으로 맞춰** 줍니다(아래 여백·이중 스크롤 방지). '쉽게 보기' 토글 등으로 내용 높이가 바뀌어도 다시 맞춰집니다.
- 스크립트가 막히는 스킨이라도 `min-height:600px` 덕분에 **고정 높이(내부 스크롤)로 자연 폴백**되어 도구는 정상 동작합니다.
- `?from=tistory` 는 유입 구분용 표식으로, 현재는 도구 동작에 영향 없습니다(향후 방문자 카운터 연동 대비).
- 폭 100% 반응형이며, 정적 파일이라 트래픽이 몰려도 서버 부하가 없습니다.

## 배포 ③ 네이버 블로그용 링크 유도 문구 (도구 삽입 X)
네이버 블로그 본문은 iframe이 불가하므로 **도구를 넣지 않고**, 소개글 + CTA 버튼/링크만 둡니다.
링크 대상은 **티스토리 해석기 글 URL**(배포 ②를 붙여넣은 글)이며 github.io 주소가 아닙니다.

- 링크 대상 예: `https://<블로그>.tistory.com/<글번호>`
- (선택) 유입 구분이 필요하면 뒤에 캠페인 파라미터를 붙일 수 있습니다: `https://<블로그>.tistory.com/<글번호>?ref=naver`

바로 쓸 수 있는 소개글 + 버튼 문구 예시:
- "등기부등본, 어렵게 보지 마세요. 아래에서 무료로 쉽게 정리해 드립니다 → **[등기부 쉽게 보기]**"
- "복잡한 갑구·을구, 시간순으로 한눈에. 설치 없이 브라우저에서 바로 → **[해석기 사용하기]**"
- "내 등기부, 뭐가 적혀 있는지 쉬운 말로 정리해 보기 → **[무료로 확인하기]**"

(위 **[ ]** 부분에 티스토리 글 URL을 링크로 겁니다.)

## 도메인 잠금 (직접 접속 → 티스토리 유도)
github.io 주소로 **직접 접속하면 파싱이 잠기고**, "블로그 글에서 이용하세요" 안내와 티스토리 글로 가는 버튼이 표시됩니다.
**티스토리 글 iframe 안(배포 ②)에서는 정상 작동**합니다. 방문자의 체류시간을 티스토리 글에 귀속시키기 위한 장치입니다.

### 설정 (`js/app.js` 상단 `ACCESS` 블록)
| 항목 | 설명 |
|---|---|
| `tistoryPostUrl` | 안내 화면의 이동 버튼이 가리킬 **티스토리 해석기 글 URL**. **글을 발행한 뒤 그 URL을 입력**하세요(예: `"https://내블로그.tistory.com/123"`). 기본값 `""`(비어 있음)이면 버튼 없이 안내 문구만 표시됩니다. |
| `lockEnabled` | 잠금 전체 켜기/끄기(`true`/`false`). `false`면 직접 접속도 그대로 사용 가능합니다. |
| `devKey` | 운영자 해제 키. `https://…github.io/…/?dev=키` 로 접속하면 운영자 본인은 직접 접속해도 사용할 수 있습니다(브라우저 세션 동안 유지 — 이후 같은 탭에서는 키 없이도 사용 가능). |
| `allowedParents` | 허용 부모 도메인 목록(기본 `["tistory.com"]`). 이 도메인 글의 iframe 안에서만 파싱이 허용됩니다. |

### 잠금이 풀리는(정상 작동하는) 경우
- 티스토리 글 iframe 안에서 열렸을 때(`allowedParents`)
- `localhost` 로컬 테스트
- `?dev=키` 로 접속한 운영자(세션 유지)
- `lockEnabled: false` 로 잠금을 끈 경우

### 운영 순서
1. 티스토리에 해석기 글 발행(배포 ② 코드 삽입) → 글 URL 확보.
2. `js/app.js`의 `ACCESS.tistoryPostUrl`에 그 URL 입력.
3. 재배포(업데이트 배포 캐시 주의 — 위 배포 절 참고). 직접 접속으로 안내 페이지가 뜨는지, `?dev=키`로 해제되는지 확인.

### 한계 (정직 고지)
referrer 위조·차단, 개발자도구 이용 등으로 **100% 차단은 아닙니다.** 일반 방문자 수준의 우회(주소 직접 공유·직접 접속)를 막아 체류를 티스토리에 모으는 장치이며, 보안 기능이 아닙니다.
